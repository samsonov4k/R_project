# Проект "Анализ покупок по карте лояльности продуктового магазина"

Автор Александр Самсонов, Владивосток

Цель проекта - анализ сложных данных на R, классификация товаров, вывод графиков.

В исследования взяты покупки по одной личной карте одного магазина. 

Для исследования из исходного датасета берутся колонки «Дата», «Наименование», «Количество», «Сумма», остальные колонки первоисточника игнорируются. Исходный датасет имеет формат tsv с разделителем табуляция и разделителем дробной части запятая.

В data/purchases2.tsv лежит сокращённый вариант датасета только по двум самым маленьким категориям «Кондитерские изделия» и «Яйца», в котором содержатся реальные покупки за 3 года.

Данная работа может иметь практический интерес с целью исследования конкретных покупок по вашему датасету. Кроме того, в работе примерена оригинальная схема классификации товаров на категории и подкатегории, которая может быть интересна исследователю.

### Принцип классификации товаров

Классификация товаров проводится в 3 этапа.
* Первый этап — подкатегории формируются по первому слову в названии товара
* Второй этап — подкатегории корректируются по правилу, указанному в текстовой строке в скрипте. Таким образом корректируется несколько процентов от всех наименований товаров.
* Третий этап — все подкатегории привязываются к категориям по правилу, указанному в текстовой строке в скрипте.

Классификация товаров происходит полностью в первом скрипте clustering.R
Синтаксис правил максимально приближен к естественному языку и описан в комментариях внутри скрипта.

## Файлы проекта:

* clustering.R
* stacked_diag.R
* one_product_diag.R



### Скрипт **clustering.R**

Скрипт **clustering.R** читает исходный файл с покупками, формирует из колонки `Наименование` колонки `Номенклатура` и `Характеристики`, а затем выполняет кластеризацию товаров с выделением колонок `Подкатегория` и `Категория`.

В результате работы скрипта формируются следующие файлы:

* **purchases_clustered.tsv** — основной датасет с добавленными кластерными признаками;
* **clusters.tsv** — таблица с результатами кластеризации;
* **part.csv** — сокращённая выборка для демонстрации возможностей скрипта.

Также скрипт строит treemap по покупкам за весь период, показывая распределение суммы по категориям.



### Скрипт **stacked_diag.R**

Скрипт предназначен для визуализации всех расходов за период. Применена группировка по колонке Категория.
Используется график "Областная диаграмма"

### Скрипт **one_product_diag.R**

Данный скрипт формирует график изменения одной цены на примере товара "Яйцо отборное"
